# Thinking

A collection of system prompts for AI that's built to expand human thinking rather than replace it. Each prompt resists the default gravity of an LLM — to converge, resolve, and answer — in favor of widening the aperture: multiplying perspective, surfacing what's invisible to the person holding it, or making a reasoning process visible and lived rather than lectured.

The organizing test for anything in this collection: **what does it hold open, and what does it refuse to close?** A tool that answers that question is a member of this family. A tool that just answers things well, however helpfully, isn't — that's a different and equally legitimate thing, just not this thing.

## The three folders

- **`outward/`** — tools that multiply perspective onto something the user brings: an idea, a decision, a plan. The user stays the subject; the multiplication happens around what they've made or proposed.
- **`inward/`** — tools that multiply perspective onto the user themselves: their voice, their beliefs, their contradictions, their unexamined patterns. Nothing here hands the user a finished verdict about who they are — every tool in this folder is built to reflect a pattern back and let the user confirm, correct, or reject it.
- **`process/`** — tools that borrow a real, named pedagogical structure (Socratic Circle, Think-Pair-Share, Cognitive Apprenticeship, and others) and hold to that structure's actual mechanics rather than a generic gloss of it. Several of these ship with a fidelity checklist alongside the prompt, since "faithful to the pedagogy" is a checkable claim, not just a description.

## A shared failure mode worth knowing about

Every prompt in this collection is fighting the same underlying pull, surfacing at a different specific moment depending on the tool: the AI quietly doing the cognitive work the human was supposed to do. In Board of Advisors, that's the synthesis converging into a recommendation when it should map tension instead. In Cognitive Apprenticeship, that's Coaching relapsing into Modeling. In Contradiction Mapping, that's "holding the tension" sliding into a quiet resolution. The specific guardrails differ by prompt, but the shape of the failure is the same family resemblance running underneath all of them — worth keeping in mind when testing any new addition to this set, since it's the first place to look when something doesn't feel right in a session.

## Status

Everything here is a drafted system prompt, some not yet validated against real conversations. Each has a fidelity checklist in `checklists/`. **Not all have been run against a real transcript yet** — the checklists are _designed-for-verification_, not verification already done They exist so a real session can be scored against specific, falsifiable items, and to give a baseline to test the prompts against: where a real session fails an item, either the session drifted or the item is wrong. Both are useful findings. The next phase is full human-in-the-loop testing.
